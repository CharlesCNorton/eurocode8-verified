# eurocode8-verified — roadmap

1. Correct standard citation from EN 1998-1-1:2024 to EN 1998-1:2004 throughout (the spectrum tables, q0 Table 5.1, DCL/DCM/DCH nomenclature, drift limits, and importance factors all come from the 2004 edition; the 2024 second-generation Eurocode 8 replaced DCL/DCM/DCH with DC1/DC2/DC3 and decomposed the behavior factor into qR·qS·qD)
2. Fix q0 for DCM DualWallEquiv and DCM DuctileWallSystem to use `3 * au_a1` instead of hardcoded `3` (EN 1998-1:2004 Table 5.1 gives 3·αu/α1 for wall-equivalent dual systems and coupled ductile wall systems in DCM; the current values are incorrect)
3. Add a `T >= 0` guard to the Se definition (currently Se accepts negative periods and returns a physically meaningless negative spectral acceleration when T < 0 and TB > 0; either add an explicit `Rmax T 0` wrapper or use a dependent type constraint)

4. Delete the generated eurocode8.mli (it is incompatible with `Extract Inlined Constant R => "float"` — the .mli declares `RbaseSymbolsImpl : RbaseSymbolsSig` with `coq_R`, `coq_R0`, etc. which no longer exist in the .ml after inlining)
5. Add `*.mli` to .gitignore (the .mli is auto-generated by Coq extraction and should not be committed)
6. Fix Makefile to delete .mli before OCaml compilation and to work with platform-appropriate compiler paths (currently hardcodes `ocamlopt` which is not on PATH; native compilation also requires mingw-gcc on Windows)
7. Prune dead constructive-reals infrastructure (~500 lines of cReal, dReal, dRealQlim, lowerCutBelow, lowerCutAbove, rational arithmetic, Z module, positive arithmetic) from extracted OCaml — none of it is reachable at runtime because R is inlined to float, and the `sig_forall_dec` stub hits `assert false` on any path through it

8. Add an epsilon-tolerant float comparison utility for extracted-code validation (structural float equality via `Req_EM_T` is fragile under different arithmetic evaluation paths)
9. Convert test_eurocode8.ml to assertion-based checks that fail on mismatch, not just print expected vs actual
10. Add failing test cases for each individual compliance check (eccentricity violation, drift violation, P-delta violation, mass ratio violation, stiffness discontinuity, q mismatch, bad storey data) to confirm the checker correctly rejects non-compliant inputs
11. Add a test case documenting the float equality fragility in ec8_compliant_dec (demonstrate that computing spar_q via a different arithmetic path than `q0` can cause spurious compliance failure due to IEEE 754 non-associativity)

12. Parameterize the P-delta compliance threshold in ec8_compliant (currently hardcoded to θ ≤ 0.20, which excludes the PD_Detailed regime 0.20 < θ ≤ 0.30 that the standard permits with explicit second-order analysis; add an analysis-type flag or parameterize theta_max)
13. Encode the storey height convention in types (st_z is cumulative height from base, not individual storey height; storey_data_consistent_aux starts from prev_z = 0 and requires sd_h = st_z - prev_z, so a user who provides st_z as storey height gets a silent consistency failure; add a well-formedness constraint requiring strictly increasing st_z values)
14. Prove actual continuity of Se on [0, ∞) by composing branch continuity with branch agreement at corner periods
15. Compute storey_data (dr, h, θ) from building model and seismic parameters inside ec8_compliant instead of taking them as exogenous input
16. Define η computation: η = max(√(10/(5+ξ)), 0.55) instead of taking η as a parameter
17. Add direction-dependent properties to building model (separate T1, stiffnesses, drifts for x and y directions)

18. Add compactness check to plan_regular (clause 4.2.3.2: approximately convex plan shape)
19. Add setback constraints to elevation_regular (clause 4.2.3.3)
20. Add 80%-of-average-of-three-storeys-above rule to stiffness_continuity (clause 4.2.3.3)
21. Add vertical seismic action spectrum (clause 4.3.3.5.2)
22. Add accidental torsion amplification (clause 4.3.3.2.4: multiply by 1 + 0.6·x/Le)
23. Add combination of seismic components (clause 4.3.3.5.1: √(Ex² + Ey²) or 100%+30% rules)
24. Add modal response spectrum analysis (clause 4.3.3.3)
25. Add capacity design rules (clause 5.2.3+)
26. Add site-specific spectrum support (clause 3.2.2.3)
27. Add near-fault directivity provisions
